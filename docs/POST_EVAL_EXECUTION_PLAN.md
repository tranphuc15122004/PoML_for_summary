## Phase B — validate and summarize the clean results

1. Compare the result counts against `models/data_audit/integrity_report.json`.
2. Check for failed models, empty generations, repeated-output spikes, and
   unexpected NaN metrics.
3. Treat the clean run as a new evaluation snapshot. Do not mix its numbers
   with the historical sampled snapshots in one table.
4. Produce a compact table for these canonical variants:

   - Base pretrained
   - Base + SFT
   - Base + fresh GRPO v5
   - Base + SFT + GRPO v5
   - Instruct pretrained
   - Instruct + SFT
   - Instruct + fresh GRPO v5
   - Instruct + SFT + GRPO v5

5. Report, at minimum, ROUGE-2, length distance, relative length error, and
   sample count by VietNews/WikiLingua. Add ROUGE-1/ROUGE-L and sentence
   adherence through an explicit reporting pass before submission; the legacy
   evaluator is primarily a ROUGE-2 evaluator.

6. Check whether the clean greedy results support the existing narrative:

   - SFT is the dominant gain;
   - SFT-initialized GRPO is a refinement;
   - fresh GRPO is weaker than SFT initialization;
   - v3-to-v5 safeguards reduce degenerate behavior.

   If a claim is not supported by the clean results, weaken or remove it rather
   than replacing it with a historical number.

## Phase D — update the manuscript

1. Update the main stage-wise table and all corresponding prose in
   `FormalReport_VDT/latex/acl_latex.tex`.
2. Replace abstract numbers only with verified clean-run numbers.
3. Update the Results and Conclusion sections consistently; search for every
   old value before finalizing:

   ```bash
   rg -n '29\.62|26\.35|1\.16|8\.35|28\.97|25\.59' FormalReport_VDT/latex/acl_latex.tex
   ```

4. Keep the contribution framing bounded to:

   - controlled stage-wise Qwen3-4B comparison;
   - reference-conditioned length/sentence control;
   - package-level reward-hacking mitigation;
   - clean, hash-deduplicated single-document evaluation.

5. Do not claim arbitrary user-budget control, multi-document generalization,
   external-model superiority, causal attribution of individual v3-v5
   hyperparameters, repeated-seed robustness, or human factuality validation.

6. Keep ViMs/VLSP and external baseline material hidden or appendix-only unless
   a separate leakage-free, protocol-matched experiment is completed.
